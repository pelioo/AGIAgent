# 网络搜索技能 (Web Search Skills v4.0)

## 技能概述

本技能文档针对**大陆地区网络环境**优化，精选可稳定访问的搜索引擎，为 AI Agent 提供高效的网络信息检索能力。

## ⚠️ 核心原则

> **首选必应，其次百度，辅以其他国内引擎**
>
> 以下国际引擎因大陆地区无法访问，已移除：~~Google~~、~~Google HK~~、~~DuckDuckGo~~、~~Brave~~、~~Yahoo~~、~~Startpage~~、~~Ecosia~~、~~Qwant~~

## 一、搜索引擎列表

### 可用搜索引擎（7个）

| 引擎 | 擅长领域 | 特色功能 | URL格式 |
|------|----------|----------|---------|
| **必应中国** ⭐首选 | 中英双语、国际资讯 | 搜索结果质量高 | `https://cn.bing.com/search?q={keyword}` |
| **百度** | 中文网页、综合搜索 | 百度百科、百度知道、文库 | `https://www.baidu.com/s?wd={keyword}` |
| **搜狗** | 微信内容、新闻资讯 | 微信搜索（独家）、搜狗问问 | `https://sogou.com/web?query={keyword}` |
| **360** | 安全内容、软件下载 | 360软件管家、360问答 | `https://www.so.com/s?q={keyword}` |
| **头条** | 新闻资讯、短视频 | 头条搜索、字节系内容 | `https://so.toutiao.com/search?keyword={keyword}` |
| **微信搜索** | 微信公众号文章 | 搜狗独家微信搜索 | `https://wx.sogou.com/weixin?type=2&query={keyword}` |
| **Jisilu** | 量化投资、社区讨论 | 集思录投资社区 | `https://www.jisilu.cn/explore/?keyword={keyword}` |

### 特殊查询引擎（1个）

| 引擎 | 擅长领域 | URL格式 |
|------|----------|---------|
| **WolframAlpha** | 数学计算、知识查询 | `https://www.wolframalpha.com/input?i={keyword}` |

## 三、搜索语法

### 3.1 必应搜索语法（首选）

```javascript
// 基础搜索
web_fetch({"url": "https://cn.bing.com/search?q=人工智能"})

// 站内搜索
web_fetch({"url": "https://cn.bing.com/search?q=site:github.com+python"})

// 精确匹配
web_fetch({"url": "https://cn.bing.com/search?q=\"机器学习\""})

// 排除词
web_fetch({"url": "https://cn.bing.com/search?q=java -coffee -script"})

// 文件类型
web_fetch({"url": "https://cn.bing.com/search?q=深度学习+filetype:pdf"})

// 组合搜索
web_fetch({"url": "https://cn.bing.com/search?q=python+tutorial+入门"})

// 新闻搜索

// 图片搜索
```

## 五、高级操作符

### 5.1 通用操作符（必应/百度支持）

| 操作符 | 功能 | 示例 |
|--------|------|------|
| `" "` | 精确匹配 | `"机器学习"` |
| `-` | 排除关键词 | `python -snake` |
| `site:` | 站内搜索 | `site:github.com python` |
| `OR` | 或运算 | `machine learning OR deep learning` |
| `filetype:` | 文件类型 | `filetype:pdf report` |

### 5.2 百度专有操作符

| 操作符 | 功能 | 示例 |
|--------|------|------|
| `intitle:` | 标题包含 | `intitle:Python教程` |
| `inurl:` | URL包含 | `inurl:login` |

### 5.3 时间过滤参数

| 参数 | 描述 | 适用引擎 |
|------|------|----------|
| `tbs=qdr:h` | 过去一小时 | 必应 |
| `tbs=qdr:d` | 过去一天 | 必应 |
| `tbs=qdr:w` | 过去一周 | 必应 |
| `tbs=qdr:m` | 过去一月 | 必应 |

## 八、搜索结果验证机制

### 8.1 可信度检查清单

搜索结果获取后，建议按照以下清单进行验证：

#### 来源验证
- [ ] 是否来自权威网站（官方文档、知名媒体、学术机构）
- [ ] 作者是否有相关资质或认证
- [ ] 是否有明确的发布时间
- [ ] 网站是否有完善的关于我们/联系方式

#### 内容交叉验证
- [ ] 关键信息是否在多个来源中得到确认
- [ ] 数据是否有官方出处标注
- [ ] 观点是否有具体证据或数据支撑
- [ ] 是否存在明显的信息矛盾

#### 时效性验证
- [ ] 内容是否为最新发布
- [ ] 技术文档是否与当前版本匹配
- [ ] 法规政策是否有最新更新
- [ ] 联系方式/价格等信息是否过期

#### 专业性验证
- [ ] 技术术语使用是否准确
- [ ] 逻辑推理是否严谨
- [ ] 是否存在明显的利益相关（广告/软文）

### 8.2 多源验证策略

```javascript
// 关键信息多源验证示例

// 场景：验证 Python Web 框架性能数据

// 源2：技术博客
```

## 十、最佳实践

### 搜索前
1. **明确搜索目标**：技术问题/学术资料/实时数据/社交内容
2. **选择合适引擎**：首选必应，复杂中文用百度
3. **构造精准关键词**：具体化、避免口语化

### 搜索中
1. **优先站内搜索**：使用 `site:` 提高精度
2. **使用精确匹配**：减少噪声干扰
3. **组合多个关键词**：提高召回率

### 搜索后
1. **优先查看权威来源**：官方文档、知名社区
2. **交叉验证重要信息**：多源对比
3. **复杂问题多引擎对比**：不同引擎互补
